# Spec: PR 1 ‚Äî Oracle Schema Foundation (Sequences + PL/SQL Packages)

**Change ID**: `oracle-stored-procedures`
**PR slice**: `pr1-oracle-schema`
**File governed**: `database/oracle_schema.sql`

---

## 1. Scope

This spec defines what MUST be true about `database/oracle_schema.sql` after PR 1 is merged.
It does NOT specify implementation internals beyond what is observable and verifiable by a reviewer running the script against a clean or existing `BIBLIOTECA` schema.

---

## 2. Requirements

### 2.1 Sequences

**R-SEQ-01** ‚Äî `oracle_schema.sql` MUST create all seven sequences listed below, each in the `BIBLIOTECA` schema:

| Sequence name             | Owned-by column              |
|---------------------------|------------------------------|
| `seq_roles_id`            | `BIBLIOTECA.roles.id`        |
| `seq_library_users_id`    | `BIBLIOTECA.library_users.id`|
| `seq_categories_id`       | `BIBLIOTECA.categories.id`   |
| `seq_authors_id`          | `BIBLIOTECA.authors.id`      |
| `seq_books_id`            | `BIBLIOTECA.books.id`        |
| `seq_loans_id`            | `BIBLIOTECA.loans.id`        |
| `seq_returns_id`          | `BIBLIOTECA.returns.id`      |

**R-SEQ-02** ‚Äî Every sequence MUST be created with `START WITH 1 INCREMENT BY 1 NOCACHE NOCYCLE`.

**R-SEQ-03** ‚Äî Every sequence creation MUST be wrapped in an idempotency guard that swallows `ORA-00955` (SQLCODE = -955) and re-raises all other exceptions. Using bare `WHEN OTHERS THEN NULL` is forbidden.

**R-SEQ-04** ‚Äî Existing `GENERATED BY DEFAULT AS IDENTITY` column definitions MUST NOT be altered. Sequences coexist with IDENTITY; procedures are authoritative for ID assignment, IDENTITY is kept for backward compatibility only.

**R-SEQ-05** ‚Äî A comment MUST appear in the schema header documenting the dual-ownership rule: "stored procedures assign PKs via sequences; IDENTITY columns are preserved for ad-hoc development inserts only."

### 2.2 Packages ‚Äî General Rules

**R-PKG-01** ‚Äî Seven packages MUST exist in `BIBLIOTECA`: `pkg_roles`, `pkg_library_users`, `pkg_categories`, `pkg_authors`, `pkg_books`, `pkg_loans`, `pkg_returns`. Each MUST have a package spec (`CREATE OR REPLACE PACKAGE`) and a package body (`CREATE OR REPLACE PACKAGE BODY`).

**R-PKG-02** ‚Äî `CREATE OR REPLACE PACKAGE` and `CREATE OR REPLACE PACKAGE BODY` MUST be used for all packages, making them idempotent by Oracle semantics.

**R-PKG-03** ‚Äî Every package MUST expose at minimum the three standard procedures: `p_insert`, `p_update`, `p_delete` with the signatures defined per-aggregate below.

**R-PKG-04** ‚Äî `p_insert` in every package MUST include an `OUT` parameter `p_id OUT NUMBER` that receives the sequence NEXTVAL value assigned to the new row.

**R-PKG-05** ‚Äî No package procedure MUST duplicate logic already owned by an existing trigger (`trg_loans_checkout_stock`, `trg_returns_restore_stock`, `trg_books_updated_at`, etc.). Trigger-owned side effects (stock decrement/restore, `updated_at`) are implicit.

**R-PKG-06** ‚Äî Package procedure bodies MUST NOT use `DBMS_OUTPUT`. Diagnostic feedback MUST use OUT parameters or exceptions only.

### 2.3 Package: `pkg_roles`

**R-PKG-ROLES-01** ‚Äî `p_insert(p_name IN VARCHAR2, p_id OUT NUMBER)` MUST assign `p_id := seq_roles_id.NEXTVAL` and INSERT a row into `BIBLIOTECA.roles`.

**R-PKG-ROLES-02** ‚Äî `p_update(p_id IN NUMBER, p_name IN VARCHAR2)` MUST UPDATE the matching row in `BIBLIOTECA.roles`.

**R-PKG-ROLES-03** ‚Äî `p_delete(p_id IN NUMBER)` MUST DELETE the matching row from `BIBLIOTECA.roles`.

### 2.4 Package: `pkg_library_users`

**R-PKG-USERS-01** ‚Äî `p_insert(p_username IN VARCHAR2, p_email IN VARCHAR2, p_password_hash IN VARCHAR2, p_role_id IN NUMBER, p_id OUT NUMBER)` MUST assign `p_id := seq_library_users_id.NEXTVAL` and INSERT into `BIBLIOTECA.library_users`.

**R-PKG-USERS-02** ‚Äî `p_update(p_id IN NUMBER, p_username IN VARCHAR2, p_email IN VARCHAR2, p_password_hash IN VARCHAR2, p_role_id IN NUMBER)` MUST UPDATE the matching row.

**R-PKG-USERS-03** ‚Äî `p_delete(p_id IN NUMBER)` MUST DELETE the matching row.

### 2.5 Package: `pkg_categories`

**R-PKG-CAT-01** ‚Äî `p_insert(p_name IN VARCHAR2, p_description IN VARCHAR2, p_id OUT NUMBER)` MUST assign `p_id := seq_categories_id.NEXTVAL` and INSERT into `BIBLIOTECA.categories`.

**R-PKG-CAT-02** ‚Äî `p_update(p_id IN NUMBER, p_name IN VARCHAR2, p_description IN VARCHAR2)` MUST UPDATE the matching row.

**R-PKG-CAT-03** ‚Äî `p_delete(p_id IN NUMBER)` MUST DELETE the matching row.

### 2.6 Package: `pkg_authors`

**R-PKG-AUTH-01** ‚Äî `p_insert(p_first_name IN VARCHAR2, p_last_name IN VARCHAR2, p_bio IN VARCHAR2, p_id OUT NUMBER)` MUST assign `p_id := seq_authors_id.NEXTVAL` and INSERT into `BIBLIOTECA.authors`.

**R-PKG-AUTH-02** ‚Äî `p_update(p_id IN NUMBER, p_first_name IN VARCHAR2, p_last_name IN VARCHAR2, p_bio IN VARCHAR2)` MUST UPDATE the matching row.

**R-PKG-AUTH-03** ‚Äî `p_delete(p_id IN NUMBER)` MUST DELETE the matching row.

### 2.7 Package: `pkg_books`

**R-PKG-BOOKS-01** ‚Äî `p_insert(p_title IN VARCHAR2, p_isbn IN VARCHAR2, p_publication_year IN NUMBER, p_stock_total IN NUMBER, p_stock_available IN NUMBER, p_category_id IN NUMBER, p_id OUT NUMBER)` MUST assign `p_id := seq_books_id.NEXTVAL` and INSERT into `BIBLIOTECA.books`.

**R-PKG-BOOKS-02** ‚Äî `p_update(p_id IN NUMBER, p_title IN VARCHAR2, p_isbn IN VARCHAR2, p_publication_year IN NUMBER, p_stock_total IN NUMBER, p_stock_available IN NUMBER, p_category_id IN NUMBER)` MUST UPDATE the matching row.

**R-PKG-BOOKS-03** ‚Äî `p_delete(p_id IN NUMBER)` MUST DELETE the matching row and MUST also DELETE all rows in `BIBLIOTECA.book_authors` where `book_id = p_id` before deleting the book (FK cascade or explicit delete).

**R-PKG-BOOKS-04** ‚Äî `p_add_author(p_book_id IN NUMBER, p_author_id IN NUMBER)` MUST INSERT a row into `BIBLIOTECA.book_authors`. MUST be a no-op (or silently succeed) if the pair already exists.

**R-PKG-BOOKS-05** ‚Äî `p_remove_author(p_book_id IN NUMBER, p_author_id IN NUMBER)` MUST DELETE the matching row from `BIBLIOTECA.book_authors` if it exists.

**R-PKG-BOOKS-06** ‚Äî `p_clear_authors(p_book_id IN NUMBER)` MUST DELETE all rows from `BIBLIOTECA.book_authors` where `book_id = p_book_id`.

**R-PKG-BOOKS-07** ‚Äî `p_set_authors(p_book_id IN NUMBER, p_author_ids IN SYS.ODCINUMBERLIST)` is OPTIONAL for this PR; if present it MUST call `p_clear_authors` then iterate inserting each author. If absent, the Python layer calls `p_clear_authors` + `p_add_author` in a loop (per ADR-5).

### 2.8 Package: `pkg_loans`

**R-PKG-LOANS-01** ‚Äî `p_insert(p_user_id IN NUMBER, p_book_id IN NUMBER, p_loan_date IN DATE, p_due_date IN DATE, p_id OUT NUMBER)` MUST assign `p_id := seq_loans_id.NEXTVAL` and INSERT into `BIBLIOTECA.loans` with `status = 'ACTIVE'`. It MUST NOT modify `books.stock` ‚Äî the existing `trg_loans_checkout_stock` trigger owns that side effect.

**R-PKG-LOANS-02** ‚Äî `p_update(p_id IN NUMBER, p_due_date IN DATE, p_status IN VARCHAR2)` MUST UPDATE the matching row. `p_status` MUST be validated against the allowed set `('ACTIVE', 'RETURNED', 'OVERDUE', 'CANCELLED')` and raise an application error if invalid.

**R-PKG-LOANS-03** ‚Äî `p_delete(p_id IN NUMBER)` MUST DELETE the matching row from `BIBLIOTECA.loans`.

**R-PKG-LOANS-04** ‚Äî `p_cancel(p_loan_id IN NUMBER)` MUST:
- Update `loans.status = 'CANCELLED'` for the given loan.
- NOT modify `books.stock` (cancellation is not a return; stock restoration is owned by `trg_returns_restore_stock`).
- Raise `APPLICATION_ERROR(-20001, ...)` if the current status is already `'RETURNED'` or `'CANCELLED'`.

### 2.9 Package: `pkg_returns`

**R-PKG-RET-01** ‚Äî `p_process(p_loan_id IN NUMBER, p_return_date IN DATE, p_fine_amount IN NUMBER, p_notes IN VARCHAR2, p_id OUT NUMBER)` MUST assign `p_id := seq_returns_id.NEXTVAL` and INSERT a row into `BIBLIOTECA.returns`. It MUST NOT modify `loans.status` or `books.stock` directly ‚Äî the existing `trg_returns_restore_stock` trigger owns those side effects.

### 2.10 Single-Source-of-Truth

**R-SSOT-01** ‚Äî `database/oracle_schema.sql` MUST remain the single DDL source of truth. No additional `.sql` files for sequences or packages SHALL be introduced unless explicitly approved by the tech lead.

## 3. Acceptance Scenarios

### SCENARIO-SEQ-01 ‚Äî Schema applies idempotently to a fresh schema

**Given** a clean `BIBLIOTECA` schema with only tables, triggers, and FKs
**When** `oracle_schema.sql` is executed via SQLPlus
**Then** all seven sequences are created without error
**And** all seven packages (spec + body) are created without error
**And** the script exits with 0 errors reported

### SCENARIO-SEQ-02 ‚Äî Schema applies idempotently to an existing schema

**Given** a `BIBLIOTECA` schema where PR 1 has already been applied
**When** `oracle_schema.sql` is executed a second time
**Then** the script completes without error (ORA-00955 swallowed for sequences, `CREATE OR REPLACE` for packages)
**And** sequence values are NOT reset

### SCENARIO-SEQ-03 ‚Äî Non-955 errors are not swallowed

**Given** the idempotency block for a sequence creation
**When** Oracle raises any error other than ORA-00955 (e.g., ORA-01031 insufficient privileges)
**Then** the error propagates and the script fails visibly

### SCENARIO-PKG-INS-01 ‚Äî p_insert returns a new ID via OUT param

**Given** `pkg_books` is deployed and `seq_books_id` exists with current value N
**When** `pkg_books.p_insert` is called with valid book data
**Then** a new row is inserted into `books` with `id = N+1`
**And** `p_id OUT` returns `N+1`
**And** `seq_books_id.CURRVAL = N+1`

### SCENARIO-PKG-INS-02 ‚Äî p_insert for loans does not touch stock

**Given** a book with `stock_available = 2`
**When** `pkg_loans.p_insert` is called for that book
**Then** a loan row is inserted with `status = 'ACTIVE'`
**And** the existing trigger `trg_loans_checkout_stock` decrements `stock_available` to 1
**And** `pkg_loans.p_insert` itself contains no explicit stock UPDATE

### SCENARIO-PKG-CANCEL-01 ‚Äî p_cancel sets status without restoring stock

**Given** a loan in `ACTIVE` status with the associated book at `stock_available = 1`
**When** `pkg_loans.p_cancel` is called with that loan's ID
**Then** `loans.status = 'CANCELLED'`
**And** `books.stock_available` remains 1 (unchanged)

### SCENARIO-PKG-CANCEL-02 ‚Äî p_cancel raises on already-terminal status

**Given** a loan with `status = 'RETURNED'`
**When** `pkg_loans.p_cancel` is called
**Then** an application error (`APPLICATION_ERROR -20001`) is raised
**And** no DML is performed

### SCENARIO-PKG-CANCEL-03 ‚Äî p_cancel raises on already-cancelled status

**Given** a loan with `status = 'CANCELLED'`
**When** `pkg_loans.p_cancel` is called again
**Then** an application error is raised

### SCENARIO-PKG-RET-01 ‚Äî p_process delegates side effects to trigger

**Given** a loan in `ACTIVE` status with book `stock_available = 1`
**When** `pkg_returns.p_process` is called with that loan's ID
**Then** a row is inserted into `returns`
**And** the existing trigger `trg_returns_restore_stock` sets `loans.status = 'RETURNED'`
**And** the trigger increments `books.stock_available` to 2
**And** `pkg_returns.p_process` itself contains no explicit UPDATE on `loans` or `books`

### SCENARIO-PKG-BOOKS-AUTHORS-01 ‚Äî p_clear_authors + p_add_author manage the link table

**Given** a book with 2 existing authors in `book_authors`
**When** `p_clear_authors(book_id)` is called
**Then** all rows for that book are deleted from `book_authors`
**When** `p_add_author(book_id, new_author_id)` is called
**Then** a new row is inserted into `book_authors`

### SCENARIO-SSOT-01 ‚Äî No duplicate DDL files

**Given** the repository after PR 1 is merged
**When** listing all `.sql` files
**Then** sequences and package DDL exist only in `database/oracle_schema.sql`

## 4. Rollback Plan

If PR 1 must be reverted after being applied to a deployed database:

1. `git revert <pr1-sha>` removes the additions from `oracle_schema.sql` in the repository.
2. The following manual cleanup script MUST be run on any deployed database:
   ```sql
   DROP PACKAGE BIBLIOTECA.pkg_returns;
   DROP PACKAGE BIBLIOTECA.pkg_loans;
   DROP PACKAGE BIBLIOTECA.pkg_books;
   DROP PACKAGE BIBLIOTECA.pkg_authors;
   DROP PACKAGE BIBLIOTECA.pkg_categories;
   DROP PACKAGE BIBLIOTECA.pkg_library_users;
   DROP PACKAGE BIBLIOTECA.pkg_roles;
   DROP SEQUENCE BIBLIOTECA.seq_returns_id;
   DROP SEQUENCE BIBLIOTECA.seq_loans_id;
   DROP SEQUENCE BIBLIOTECA.seq_books_id;
   DROP SEQUENCE BIBLIOTECA.seq_authors_id;
   DROP SEQUENCE BIBLIOTECA.seq_categories_id;
   DROP SEQUENCE BIBLIOTECA.seq_library_users_id;
   DROP SEQUENCE BIBLIOTECA.seq_roles_id;
   ```
3. Tables, triggers, FK constraints, and IDENTITY columns are untouched by this rollback.

## 5. Out of Scope

- Python repository classes (PR 2)
- pytest test suite (PR 3)
- Altering existing trigger logic
- Any `SYS_REFCURSOR`-based read procedures (reads are owned by SQLAlchemy ORM in PR 2)
- Data migrations
