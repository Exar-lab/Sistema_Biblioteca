Arquitectura general

Esquema BIBLIOTECA en Oracle 21c XE (XEPDB1)

Todo el acceso a datos pasa por PL/SQL packages. La capa Python (SQLAlchemy + cx_Oracle) nunca emite SQL directo contra las tablas — llama a los stored procedures de cada package, que retornan SYS_REFCURSOR para consultas y OUT parameters para escrituras.

Sequence7Una por cada tabla principal (gestión idempotente vía p_ensure_sequence)
Index7Sobre columnas de búsqueda frecuente (título, categoría, nombre, rol, estado de préstamo)

Packages y Procedures

Toda la lógica de negocio está encapsulada en 6 packages PL/SQL

Convención de nombres: todos los packages siguen el patrón pkg_<entidad> y sus procedures usan prefijos: p_insert, p_update, p_delete, p_sel_by_id, p_list. Las consultas retornan SYS_REFCURSOR; las escrituras retornan el nuevo id como parámetro OUT.

pkg_roles

Gestión de roles del sistema
p_insertCrea un rol. Retorna p_id OUT.
p_updateActualiza nombre y descripción por ID.
p_deleteElimina. p_deleted OUT indica si afectó alguna fila.
p_sel_by_idRetorna un cursor con el rol por ID.
p_listRetorna cursor con todos los roles ordenados por ID.

pkg_library_users

Usuarios de la biblioteca
p_insertCrea un usuario con hash de contraseña, rol y estado activo.
p_updateActualiza todos los campos incluyendo contraseña y rol.
p_deleteElimina usuario. Retorna p_deleted.
p_sel_by_idBusca usuario por ID (cursor).
p_listLista todos los usuarios.

pkg_categories

Categorías de libros
p_insertCrea categoría con nombre, descripción e is_active.
p_updateModifica nombre, descripción y estado activo.
p_deleteElimina si no tiene libros asociados (FK constraint).
p_sel_by_idRetorna cursor con la categoría.
p_listLista todas las categorías.

pkg_authors

Autores de libros
p_insertRegistra autor con nombre, biografía y fechas de vida.
p_updateModifica todos los campos del autor.
p_deleteElimina (CASCADE elimina también book_authors).
p_sel_by_idBusca por ID.
p_listLista todos los autores.

pkg_books

Catálogo de libros + relación autores
p_insertCrea libro con todos sus metadatos y stock inicial.
p_updateActualiza metadatos y stock.
p_deleteElimina libro (CASCADE en book_authors).
p_add_authorAsocia un autor al libro. Ignora duplicados (ORA-00001).
p_remove_authorDesasocia un autor específico.
p_set_authorsReemplaza todos los autores del libro con SYS.ODCINUMBERLIST.

pkg_loans

Préstamos — con control de stock
p_insertCrea préstamo ACTIVE. El trigger trg_loans_checkout_stock descuenta el stock automáticamente.
p_updateModifica campos del préstamo (uso interno / admin).
p_cancelCancela préstamo ACTIVE y restaura stock. Lanza error si ya está RETURNED o CANCELLED.
p_sel_by_idRetorna cursor con préstamo por ID.
p_listLista todos los préstamos.

pkg_returns

Devoluciones — trigger se encarga del stock
p_processInserta un registro de devolución. El trigger trg_returns_restore_stock actualiza el préstamo a RETURNED y restaura el stock.
p_updateModifica notas y monto de multa (admin).
p_deleteElimina registro de devolución.
p_sel_by_idBusca devolución por ID.
p_listLista todas las devoluciones.

Sequences

Generadores de IDs — gestionadas de forma idempotente

Procedimiento auxiliar p_ensure_sequence: en vez de crear las sequences con un CREATE SEQUENCE simple, el schema usa este procedimiento que: (1) crea si no existe, (2) si existe, avanza el valor al MAX(id) + 1 de la tabla para evitar conflictos al re-ejecutar el script de inicialización.

Flujo: Crear préstamo

Desde la petición HTTP hasta el commit en Oracle

1

Frontend → API

El usuario (autenticado) hace POST /api/loans con book_id, loan_date y due_date. La sesión Flask aporta el user_id.

2

Python llama al package

El repositorio de Python llama a pkg_loans.p_insert(p_user_id, p_book_id, p_loan_date, p_due_date, p_id OUT).

3

INSERT en LOANS

El procedure inserta en BIBLIOTECA.loans con status ACTIVE. Esto dispara el trigger trg_loans_checkout_stock.

4

Trigger verifica y descuenta stock

El trigger bloquea la fila del libro (FOR UPDATE), verifica que stock_available > 0 y ejecuta UPDATE books SET stock_available = stock_available - 1. Si stock es 0, lanza error -20001.

5

COMMIT y respuesta

Python hace commit. El préstamo queda activo y el stock_available del libro decrementado en 1.

sequenceDiagram actor U as Usuario participant F as Flask API participant P as pkg_loans.p_insert participant T as trg_loans_checkout_stock participant B as BOOKS participant L as LOANS U->>F: POST /api/loans {book_id, due_date} F->>P: p_insert(user_id, book_id, loan_date, due_date) P->>L: INSERT INTO loans STATUS='ACTIVE' L-->>T: BEFORE INSERT trigger fires T->>B: SELECT stock_available FOR UPDATE alt stock_available = 0 T-->>F: RAISE_APPLICATION_ERROR -20001 F-->>U: 400 "No available stock" else stock_available > 0 T->>B: UPDATE stock_available - 1 L-->>P: RETURNING id INTO p_id P-->>F: p_id = nuevo loan_id F-->>U: 201 {loan_id} end

Flujo: Devolver libro

Proceso de devolución y restauración automática de stock

1

Frontend → API

POST /api/returns con loan_id, return_date y opcionalmente fine_amount y notes.

2

Python llama al package

El repositorio llama a pkg_returns.p_process(loan_id, return_date, fine_amount, notes, p_id OUT).

3

INSERT en RETURNS

El procedure inserta en BIBLIOTECA.returns. Esto dispara el trigger trg_returns_restore_stock (AFTER INSERT).

4

Trigger actualiza préstamo y restaura stock

Lee el book_id del préstamo, pone el préstamo en RETURNED, y hace stock_available + 1 solo si no superaría stock_total.

5

COMMIT

La devolución queda registrada, el préstamo marcado como RETURNED y el libro disponible nuevamente.

Flujo: Cancelar préstamo

Cancelación controlada vía pkg_loans.p_cancel

Importante: La cancelación no usa el trigger de devolución. p_cancel actualiza el préstamo a CANCELLED y restaura el stock directamente dentro del procedure, sin pasar por la tabla RETURNS.

Guía de ejecución del schema

Cómo inicializar o resetear la base de datos

El script es idempotente. Si la DB ya existe y se vuelve a ejecutar, las tablas y sequences no se recrean — el script detecta su existencia y las omite. Los packages sí se reemplazan (CREATE OR REPLACE).
Pasos de ejecución
1

Prerrequisito

Tener Oracle 21c XE corriendo con el PDB XEPDB1 disponible. Conectarse como SYS AS SYSDBA.

2

Ejecutar el script

Desde el directorio raíz del proyecto:
sqlplus / as sysdba @database/oracle_schema.sql
SQL*Plus pedirá una contraseña para el usuario BIBLIOTECA (se define en ese momento).

3

Configurar .env

Actualizar DB_PASSWORD en el archivo .env del proyecto con la contraseña elegida para el usuario BIBLIOTECA.

4

Verificar

El script ejecuta PROMPT BIBLIOTECA Oracle schema is ready in XEPDB1. al finalizar exitosamente. Comprobar que no haya errores ORA- en la salida.
